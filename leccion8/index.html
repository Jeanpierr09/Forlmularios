Lección 8: Validación Básica de Formularios

Validación Automática con HTML5

HTML5 introdujo una serie de atributos que permiten realizar validaciones de datos directamente en el navegador, sin necesidad de JavaScript. Esto mejora la experiencia del usuario al proporcionar feedback inmediato.

Cuando un usuario intenta enviar un formulario con campos que no cumplen estas reglas, el navegador detiene el envío y muestra un mensaje de error junto al campo incorrecto.

Atributos de Validación Comunes

  • required: El campo no puede estar vacío.
  • minlength y maxlength: Especifican la longitud mínima y máxima de caracteres para un campo de texto.
  • min y max: Definen el valor mínimo y máximo para campos numéricos (type="number") o de fecha (type="date").
  • type="email" y type="url": El navegador comprueba que el valor introducido tenga el formato de un correo electrónico o una URL, respectivamente.
  • pattern: El atributo más potente. Permite definir una expresión regular que el valor del campo debe cumplir. Es útil para formatos específicos, como un código postal o un número de teléfono.

Mensajes de Error

Los mensajes de error que aparecen son predeterminados y dependen del navegador y su idioma. Por ejemplo, "Por favor, rellena este campo" o "Introduce una dirección de correo electrónico válida". Aunque estos mensajes se pueden personalizar con JavaScript, la validación básica de HTML5 es un excelente punto de partida.

Ejemplo de Código


<form action="#">
  <div>
    <label for="username">Nombre de usuario (obligatorio, 5-10 caracteres):</label>
    <input type="text" id="username" name="username" required minlength="5" maxlength="10">
  </div>
  <div>
    <label for="email">Correo electrónico (obligatorio):</label>
    <input type="email" id="email" name="email" required>
  </div>
  <div>
    <label for="cp">Código Postal (5 dígitos):</label>
    <input type="text" id="cp" name="postal_code" pattern="[0-9]{5}" title="Debe contener 5 dígitos numéricos">
  </div>
  <button type="submit">Enviar</button>
</form>

Intenta enviar el formulario de ejemplo sin rellenar los campos o con datos incorrectos para ver la validación en acción.

Reto

Crea un formulario de registro de contraseña. Debe tener un campo de contraseña que sea obligatorio y que deba tener una longitud mínima de 8 caracteres. Añade un botón de envío para probar la validación.
